Project Overview

A cinematic local anime library

Anzuanime Server v2.0 combines Flask routes, Jinja templates, local media scanning, AniList enrichment, cached assets, and a custom player into one cohesive desktop-friendly app.

Main Features

  • Homepage featured slider, Continue Watching, Anime Collection, filters, and search.
  • Movies, Schedule, Seasons, Anime Detail, Player, and Settings pages.
  • Watch history, watch status, settings, and cache maintenance support.

Backend Features

  • SQLite library sync and local folder scanning.
  • AniList posters, banners, metadata, characters, relations, and schedule data.
  • Thumbnail, resolution, duration, subtitle, VLC, and Discord RPC integration.

Frontend/UI Features

  • Dark glass/neon visual system with cinematic page treatments.
  • Shared base layout for most pages, with special standalone player/detail layouts.
  • Responsive cards, polished badges, subtle scrollbars, and cleaner hover states.

Experience

Detail pages and playback feel app-like

Anime Detail Page Features

The anime detail page includes a large banner, poster, glass detail panel, status and metadata, genres, description, character cards, episode list, recommendations, related anime, season navigation, and watched/progress badges.

Hero Banner Characters Episode Cards Watched Badges

Video Player Features

The player uses Plyr with resume support, soft episode switching, subtitle updates, browser URL updates, active episode cards, VLC launch support, screenshots, Discord RPC, and watch progress reporting.

Plyr Subtitles VLC Screenshots

Local Control

Settings, cache, and local data

Settings Page

Settings cover watchlist, ongoing, movie, and VLC paths, plus Discord Rich Presence. Native Windows folder and file pickers make local setup easier while manual input remains available.

Cache and Local Data

The app uses cache folders for posters, banners, characters, episodes, metadata, subtitles, and thumbnails, alongside JSON files for settings, watch history, and watch status.

Watch Progress System

Watch history powers Continue Watching and resume behavior. Watch status tracks per-episode progress and watched state, then surfaces that state in anime cards and the player carousel.

Review Notes

Strengths and future polish

Strengths of v2.0

  • Strong local Windows integration and media-library workflow.
  • Polished cinematic UI with a cohesive Anzuanime identity.
  • Smooth player navigation without full reloads.
  • Practical cache-first metadata and asset system.
  • Configurable settings and conservative cache cleanup.

Possible Future Improvements

  • Split the large backend file into focused modules.
  • Add route smoke tests for core pages and APIs.
  • Add cache cleanup dry-run and local data backup/export tools.
  • Improve scanner, subtitle, cache, and RPC logging.
  • Document dependency versions and localhost-only safety expectations.
